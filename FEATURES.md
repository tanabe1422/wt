## 選定ルール

- 次に実装する候補 = `[ ]` のうち **最小の P 番号**
- 同点はドキュメント上の **上から先**
- 実装完了したら `[ ]` を `[x]` に更新する
- 部分実装は未実装のまま残し、括弧で現状を注記する
- **方針**: Git 操作の拡充を優先。ショートカット・ピン留め・テーマなど便利 / UX 系は後回し（P3）
